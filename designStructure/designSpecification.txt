智能标注校验工具（LLM驱动版）需求规格说明书

版本：3.0-Lite
最后更新：2025-07-16

---

1. 项目概述

1.1 背景与目标

传统数据标注质检依赖复杂的几何规则（如重叠率、最小面积、自相交检测等），开发成本高且难以处理语义错误（如类别混淆）。本工具旨在利用多模态大模型（VLM）的语义理解能力，替代手工编写的校验算法，实现灵活、智能的标注质量检查。

1.2 核心理念

· 去规则化：不编写固定几何校验代码，完全由VLM通过自然语言理解图像与标注的匹配度。
· 人机协同：YOLO快速预标注 + 人工微调 + VLM智能校验，保留最终确认权。
· 可定制校验标准：通过自定义Prompt，动态调整校验粒度与关注点。
· 手动触发校验：校验过程由用户主动触发，确保在人工修正后再进行校验，避免不必要的API调用。
· 舍弃管理：允许将不合格或无需迭代的图片标记为“舍弃”，并在导出时一并归档。

2. 系统角色与用例

角色 职责
标注员 加载图片、运行预标注、人工修正标注、手动触发AI校验、查看反馈、标记舍弃图片、导出数据
管理员 配置模型（YOLO路径、类别名称映射、OpenAI接口信息）、自定义校验Prompt、管理模板

3. 核心工作流（用户视角）

1. 准备阶段
   · 选择待标注图片文件夹
   · 加载本地YOLOv8模型（.pt文件）
   · 手动配置类别名称映射（将模型输出的数字索引映射为有意义的类别名称，如0→人，1→车）
   · 配置OpenAI接口：填写API Key、Endpoint（可选，默认使用OpenAI官方Endpoint）、手动输入模型名称（如gpt-4o、gpt-4-turbo等）
   · 编辑或选择校验Prompt模板
2. 预标注阶段
   · 点击“自动标注”，YOLO生成初始检测框或分割掩码，标注类别显示为配置的名称
   · 用户可拖拽、缩放、删除、添加标注（矩形/多边形）
3. 智能校验阶段
   · 触发方式：仅支持手动触发，用户完成一轮修正后，点击“AI校验”按钮发起校验
   · 处理流程：
     · 前端获取当前图片的原始图像（直接从文件读取）和叠加了标注的画布图像，分别转为Base64格式
     · 后端使用当前选定的Prompt模板，调用OpenAI API，同时传入两张图像（原图+标注图）
     · OpenAI返回文本判断结果（PASS 或 具体问题描述）
4. 反馈与修正
   · 通过：标注条目标记为绿色，允许进入下一张
   · 不通过：侧边栏展示VLM指出的问题列表，用户手动修正后可再次点击“AI校验”
5. 舍弃管理
   · 若当前图片质量差、无需标注或不应纳入训练集，用户可点击“舍弃”按钮标记该图片。
   · 被舍弃的图片不再参与后续校验流程，但保留其标注（若有），供导出时归档。
6. 导出
   · 导出所有已处理的图片（包括通过、人工确认及被舍弃的图片），按文件夹区分，便于归档。
   · 仅支持YOLO格式：每张图片对应一个同名的.txt文件，内容为class_index x_center y_center width height（归一化坐标），类别索引基于配置的映射。

4. 详细功能模块

4.1 模型配置模块

YOLO配置

· 模型文件加载：本地文件选择器，加载.pt文件
· 设备选择：支持CPU/CUDA推理
· 类别名称映射（核心功能）：
  · 由于YOLO模型输出通常仅为类别索引（0,1,2...），需要用户手动定义每个索引对应的实际类别名称，用于标注界面显示、VLM校验提示、YOLO导出等。
  · 界面提供文本编辑区，格式为每行一条：索引: 名称，例如：
    ```
    0: person
    1: bicycle
    2: car
    ```
  · “从模型读取”按钮：尝试从模型文件中自动读取类别名称（若模型保存时包含名称信息），填充到编辑区，用户可在此基础上修改。
  · “验证”按钮：检查格式是否正确（索引为数字、名称非空），并提示重复索引等问题。
  · 映射完成后，系统内部维护一个index->name的字典，所有标注操作和导出均基于此映射。

OpenAI接口配置

· API Key：输入OpenAI API密钥（必填）
· Endpoint（可选）：默认为https://api.openai.com/v1/chat/completions，可更改为代理地址
· 模型名称：手动输入文本框，用户自行输入要使用的视觉模型名称，如gpt-4o、gpt-4-turbo等，不提供下拉选择（保持最大灵活性）
· 连接测试按钮：验证配置是否成功，测试时会发送简单请求（如包含测试文本和一张简单图片）确认连通性及模型支持性

自定义Prompt编辑

· 多行文本框，支持占位符如{image}（原图）、{overlay}（标注图）和{classes}（类别名称列表）
· 提供默认Prompt模板，可恢复默认
· 支持保存多个命名模板（如“通用校验”、“行人专用”）并快速切换

4.2 标注编辑模块（基础版）

· 图形支持：矩形框、多边形（点选绘制）
· 编辑操作：移动、缩放、删除、顶点编辑（多边形）
· 类别管理：下拉选择当前类别，下拉选项来自4.1中配置的类别名称映射（显示名称，内部存储索引）
· 画布控制：缩放、平移、适应窗口

4.3 AI校验面板

· 触发按钮：显著的“AI校验”按钮，仅手动点击触发
· 状态指示：等待校验 / 校验中 / 通过 / 需修正
· 问题列表：以列表形式逐条展示VLM返回的问题描述（纯文本，不包含坐标）
· 忽略功能：对某条建议点击“忽略”，不再提示（记录忽略历史）

4.4 舍弃管理模块

· 舍弃按钮：在标注界面工具栏或图片信息栏中放置“舍弃”按钮。
· 标记状态：被舍弃的图片在缩略图列表或当前图片显示区域有明显标识（如灰色蒙版、红色斜线等）。
· 标注保留：舍弃操作不影响已有标注数据，用户仍可查看和修改标注（但通常无需再校验）。
· 状态切换：支持取消舍弃（如误操作后可恢复）。
· 过滤视图：缩略图列表可筛选显示“全部/正常/舍弃”，方便管理。

4.5 数据导出模块

· 导出范围：支持以下选项（单选或多选）：
  · 所有已通过校验的图片
  · 所有已人工确认的图片（包括未触发校验但人工确认的）
  · 所有被舍弃的图片
  · 全部图片（默认）
· 导出结构：导出为ZIP压缩包，内部按状态分文件夹存放：
  ```
  export_20250716/
  ├── passed/          # 通过校验的图片及标注
  ├── confirmed/       # 人工确认但未校验的图片及标注（可选）
  └── discarded/       # 被舍弃的图片及标注
  ```
· 格式选项：仅支持YOLO TXT格式。每张图片对应一个同名.txt文件，内容格式为：class_index x_center y_center width height（归一化坐标），类别索引基于配置的映射。
· 导出路径设置：用户选择保存位置，系统自动生成带时间戳的文件夹名。

5. 技术架构

5.1 前后端分离

· 前端：Vue3 + Canvas（渲染标注）
· 后端：FastAPI（提供API服务）

5.2 后端核心接口

接口 方法 说明
/yolo/predict POST 接收图片，返回YOLO检测结果（坐标、类别索引、置信度）
/vlm/check POST 接收原图Base64、标注图Base64和当前Prompt，调用OpenAI接口，返回校验结果（{status: "PASS"/"FAIL", issues: [...]}）
/export POST 接收导出参数（包含需要导出的状态列表），生成并返回压缩包

5.3 VLM调用逻辑（OpenAI接口，双图输入）

1. 前端准备图像：
   · 原始图像：从图片文件直接读取，转为Base64（JPEG格式，可适当压缩）
   · 标注图像：从当前画布（含标注）渲染为Base64
2. 后端接收请求：包含original_image_base64、overlay_image_base64、prompt_template、class_names等
3. Prompt占位符替换：
   · {classes} → 类别名称列表字符串
   · {image} 和 {overlay} 在Prompt文本中用于描述，但图像内容通过API的image_url传递，不嵌入文本
4. 构建OpenAI请求（支持多图像）：
   ```json
   {
     "model": "用户输入的模型名称",
     "messages": [
       {
         "role": "user",
         "content": [
           {"type": "text", "text": "这是原始图像："},
           {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,{original_base64}"}},
           {"type": "text", "text": "这是带有标注（黄色线条）的图像："},
           {"type": "image_url", "image_url": {"url": "data:image/jpeg;base64,{overlay_base64}"}},
           {"type": "text", "text": "请根据这两张图检查标注是否准确。图片中可能出现的物体类别有：{classes}。请遵循以下输出要求：..."}
         ]
       }
     ],
     "max_tokens": 300
   }
   ```
5. 解析返回：
   · 若返回文本包含“PASS”（忽略大小写），则判定为通过，问题列表为空
   · 否则，将返回文本按行分割，提取每行作为问题描述（以“FAIL:”开头或直接作为问题条目），构建问题列表
6. 返回结构化结果给前端，格式为：
   ```json
   {
     "status": "PASS" | "FAIL",
     "issues": ["问题1", "问题2", ...]  // 当status为FAIL时存在
   }
   ```

6. Prompt设计

6.1 默认Prompt模板

```
这是原始图像：
{image}
这是带有标注（黄色线条）的图像：
{overlay}

你是一个专业的数据标注校验员。图片中可能出现的物体类别有：{classes}。
请检查标注是否准确，并仅按以下要求输出：
1. 如果标注完全正确（无漏检、无误检、标注贴合、类别正确），只输出字符串："PASS"
2. 如果有问题，输出格式："FAIL: [问题描述]"，问题描述应简短明确，例如：
   - "FAIL: 左侧第二个人漏标"
   - "FAIL: 右上角的车框太大，包含背景"
   - "FAIL: 把狗标成了猫"
3. 不要输出任何其他解释或格式。
```

说明：{image}和{overlay}在文本中作为占位符，实际图像通过API的image_url字段传递，文本中的占位符可保留用于描述，但模型会自动关联图像。

6.2 自定义Prompt

· 用户可自由编辑Prompt，支持使用{classes}占位符，并可自由描述图像顺序。
· 提示：建议在Prompt中明确指明两张图的含义，以便模型理解。
· 注意：VLM仅返回文本判断，不返回坐标，因此Prompt中不应要求提供坐标信息。

自定义示例（严格模式）

```
原始图像：
{image}
标注图像（黄色线条）：
{overlay}

请严格检查图中所有黄色标注框（类别应为{classes}中的一种）：
1. 每个目标必须被框完全覆盖，不得遗漏任何部分；
2. 类别必须与物体实际种类一致；
3. 不得有多余的框（背景误检）。
若完全正确输出PASS，否则列出具体错误，每行一个"FAIL: ..."。
```

7. 操作流程（典型场景）

1. 配置：选择图片文件夹，加载YOLO模型，编辑类别映射；填写OpenAI信息，输入模型名称；编辑或选择校验Prompt。
2. 预标注：点击“自动标注”，YOLO生成初稿，标注显示为配置的名称。
3. 人工修正：调整标注。
4. 手动AI校验：点击“AI校验”，前端准备原图和标注图，发送至后端，等待返回结果。
5. 循环校验：
   · 若通过 → 标注变绿，进入下一张
   · 若不通过 → 根据问题列表手动修正，再次点击“AI校验”
6. 舍弃处理：若图片无法使用，点击“舍弃”标记。
7. 批量导出：点击“导出”，选择包含的状态（如“通过”和“舍弃”），保存到本地。

8. 优势与取舍说明

优势 取舍
无需编写复杂几何校验代码，开发周期短 依赖OpenAI API，增加延迟和成本
能识别语义错误（类别混淆、上下文错误） OpenAI偶尔产生幻觉，需人工复核
通过修改Prompt即可调整校验标准，灵活 不支持离线批量处理（除非本地模型）
手动触发节省API调用，用户掌控节奏 需人工介入触发，不适合全自动流程
支持YOLO模型仅输出索引，手动映射类别名称，适配更多模型 额外配置步骤，但一次配置可重复使用
手动输入VLM模型名称，支持新模型即时使用 用户需自行确保模型名称正确且支持多模态
舍弃功能允许归档不合格数据 简单交互，几乎无开发成本
同时传递原图和标注图，提升校验准确性 增加API调用消耗（双图token）
VLM仅返回文本判断，简化解析逻辑 无法自动定位问题区域，依赖人工查找

9. 交付标准

1. 成功对接OpenAI API，支持用户手动输入模型名称，并支持同时传递原始图像和标注图像进行校验。
2. 支持YOLOv8预标注结果的生成与编辑，并能通过手动配置将类别索引映射为有意义的名称。
3. 能够将当前图片的原图和标注图发送至OpenAI，并解析返回结果（PASS或问题列表）。
4. 界面清晰展示校验状态及问题列表，支持忽略误报。
5. 提供舍弃标记功能，支持对图片进行“舍弃”标记，并在导出时按状态分类打包。
6. 导出数据格式正确：仅支持YOLO TXT格式（类别索引），导出ZIP包内按状态分文件夹。
7. 提供Prompt模板管理功能（保存、切换、恢复默认），且支持{image}和{overlay}占位符。
8. 整体操作流程顺畅，无复杂配置，符合“精简版”定位。
